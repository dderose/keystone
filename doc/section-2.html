2.0 :: Utilities

Utility classes are isolated and targeted styles that allow us to easily modify any element from a centralized repository that avoids cascade complexity and specificty nightmares. Utility classes are meant to be sledgehammers that always take precedence and thus utilize !important.

2.1 :: Heading Sizes

These classes decouple heading styling/sizing from structure and allow us to redefine the size of any heading to match the design while keeping semantic meaning in element usage.

2.8 :: Spacing

The framework comes with the convention of standard spacing units which are equal to multiples of the vertical rhythm. One base spacing unit is therefore equal to 21.3px and there are also half and double spacing units.

These spacing units can be applied using the naming convention of u-{type}{Direction}{Size}. For example, to add double base spacing unit around an element using padding the class is u-paddingDouble.

Additionally, margin and padding can be cleared by using Flush for the size.

Finally, all spacing utilites have responsive versions of format {breakpoint}-u-{type}{Direction}{Size}. For example, b2-u-marginTopDouble and b1-u-marginTopFlush.

2.11 :: Responsive Utilities

The framework has 3 breakpoints built in. Each of those breakpoints has a variety of utilities that are only active within its specified dimensions.

The naming format for responsive utilites is: {breakpoint prefix}-u-{utility name}

Since the framework uses a mobile first approach the base utility should be used as the smallest screen size and then breakpoints should be used to target progressively larger screen sizes. For example: <div class="Grid-item b1-u-size1of2 b3-u-size1of4">Grid item</div>.

Breakpoint Prefix Dimensions
"b1-" > 481px
"b2-" > 769px
"b3-" > 1000px
"b4-" > 1280px

Additionally, the following mixins are available for use within scss when creating components. Simply call the mixin from within a css selector. For example, the following code would center align text until the screen reaches a width of 769px after which it would left align the text.

.ComponentName { text-align: center; @include rwd-b2() { text-align: left; } }.
