1.0 :: Base

Normalization, default styling for native elements

1.1 :: Normalize.css

Library URL: git.io/normalize
Normalize.css makes browsers render all elements more consistently and in line with modern standards. It precisely targets only the styles that need normalizing.

1.2 :: Reset

A thin layer on top of normalize.css that provides a starting point more suitable for web applications. Removes the default spacing and border for appropriate elements and globally sets box-sizing to border-box.

1.3 :: Typography

The base typography establishes base font family, font sizes, font colors, scale, and vertical rhythm/tempo.

The typographic scale used is the traditional scale. See http://lamb.cc/typograph/ for more information on scale and vertical rhythm/tempo.

Additionally, we use a mixin to calculate font-size that also calculates the appropriate line-height and margin-bottom to maintain the vertical rhythm/tempo. See http://twitter.com/redclov3r/status/250301539321798657.

To use the fonts in FED, an additional link must be provided with the fed link. This uses cloud.typography to determine a version of the font file specific to the OS & OS version of the user to give them the ultimate font experience. The typography link will grab a font file hosted off our servers, of which there are 40 different ones. Please do not link directly to our hosted files, as it breaks our agreement with Typography.

<link rel="stylesheet" href="//cloud.typography.com/7508852/770622/css/fonts.css" />

1.3.3 :: Base margin spacing between elements

Where margin-bottom is concerned, this value will be the same as the base line-height. This allows us to keep a consistent vertical rhythm. As per: csswizardry.com/2012/06/single-direction-margin-declarations
