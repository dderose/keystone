18.0 :: Messaging

There are multiple types of messages: info, success, warning, and error. The default (info) message uses the class Msg and modifiers are used to change it into the other types: Msg--success, Msg--warning, and Msg--error.

Additionally, Msg-heading can be applied to a heading within a message.

The correct icon is automatically added.

This is a message.

This is a success message.

This is an error message.

This is a warning message.

Message Heading

This is a succes message with a heading.

<div class="Msg">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <p class="u-marginFlush" data-qe-id="{{msg.txt}}">This is a message.</p>
</div>
<div class="Msg Msg--success">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <p class="u-marginFlush" data-qe-id="{{msg.txt}}">This is a success message.</p>
</div>
<div class="Msg Msg--error">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <p class="u-marginFlush" data-qe-id="{{msg.txt}}">This is an error message.</p>
</div>
<div class="Msg Msg--warning">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <p class="u-marginFlush" data-qe-id="{{msg.txt}}">This is a warning message.</p>
</div>
<div class="Msg Msg--success">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <h2 class="Msg-heading" data-qe-id="{{msg.heading}}">Message Heading</h2>
  <p class="u-marginFlush" data-qe-id="{{msg.txt}}">This is a succes message with a heading.</p>
</div>

18.1 :: Complex messages

The complex message style allows a more flexible format resulting in the ability to create messages such as the one below. Simply apply Msg--complex and use the Grid component to achieve the desired layout.

Success!

Buy now or call 1-888-555-1212 to keep your active campaigns running, publish new ones, and send to unlimited contacts.

<div class="Msg Msg--complex">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message" data-qe-id="{{msg.close}}"></a>
  <div class="Grid Grid--gutter u-clearfix">
    <div class="Grid-item u-sizeShrinkWrapLeft u-hide b2-u-show">
      <img src="images/demo-messaging.png" alt="" />
    </div>
    <div class="Grid-item u-sizeShrinkWrapRight">
      <button class="Button Button--secondary u-marginTopHalf" data-qe-id="{{msg.buy}}">Buy Now</button>
    </div>
    <div class="Grid-item u-sizeFill">
      <h2 class="Msg-heading" data-qe-id="{{msg.heading}}">Success!</h2>
      <p class="u-marginFlush" data-qe-id="{{msg.txt}}"><a href="#">Buy now</a> or call 1-888-555-1212 to keep your active campaigns running, publish new ones, and send to unlimited contacts.</p>
    </div>
  </div>
</div>

18.2 :: Floating messages

Floating (absolutely positioned) messages can be achieved by using the Msg--float modifier. This will also cause the message box to shrink to fit the content.

Note: Negative margin has been added only to serve to illustrate the floating nature of the element. Do not use this in prodcuction code.


This is a floating message.

<div class="Msg Msg--warning Msg--float" style="margin:-65px 0 0 33%;">
  <a href="#" class="Icon--close Msg-close" aria-role="button" aria-label="Dismiss this message"></a>
  <p class="u-marginFlush">This is a floating message.</p>
</div>
